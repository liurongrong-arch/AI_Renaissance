# 版本历史

## V4.5 | 2026-05-20 — 初始发布

**Industrial Sentinel 产业链中观分析框架**

### 核心特性

- **三维度独立展示**：产业链景气度 / 产业链拐点 / 产业链生命周期，不合成单一指数
- **真实数据驱动**：每个数字标注来源（财报/研报/新闻/机构）和时间戳，禁止算法填充
- **方法论框架**：提供透明推理规则，用户用自己的AI/Agent搜索数据填入JSON，框架自动分析
- **自动产业链检测**：输入股票代码后自动识别所属产业链，加载对应YAML卡片
- **五态拐点模型**：拐点前 / 拐点初期 / 拐点确认 / 拐点晚期 / 拐点后衰退
- **生命周期判定**：导入期 / 成长期 / 成熟期 / 衰退期
- **System B 个股类型**：成长型 / 周期型 / 价值型 / 主题型 / 混合型的决策树判定
- **产业链结构卡片**：上游/中游/下游价值分配、核心玩家、壁垒、议价权

### 已覆盖产业链

| Preset | 行业 |
|--------|------|
| `optical-module` | 光通信/光模块 |
| `ai-energy` | AI能源层 |
| `ai-chip` | AI芯片层 |
| `ai-infrastructure` | AI基础设施层 |
| `robotics` | 机器人产业链 |

### 触发词

```
/景气度 [代码]
```

### 架构

```
输入代码 → 自动检测产业链preset → 加载YAML卡片
  → 基于填入的JSON数据自动推理
    → 五态拐点判定 + 生命周期判定 + 个股类型判定
      → 输出HTML报告
```

### 数据输入

用户自行搜索财报/研报/新闻数据，填入 `data/<code>_real_data.json`，框架负责确定性推理。

详见：
- `references/data-requirements.md` — 必填数据字段清单
- `references/methodology-mapping.md` — 数据→判断映射规则
- `references/data-standardization.md` — 数据标准化指南
