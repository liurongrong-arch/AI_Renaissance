# Industrial Sentinel V4.5 — 数据标准化指南

> **目标**：减少不同用户搜索数据时的差异，确保"同样的标的 → 相似的输入数据 → 一致的输出结论"。
>
> **原则**：框架是确定性的（同样的输入→同样的输出），但数据获取是每个人自己的AI的事。本指南教你如何标准化数据搜索和填入，让不同人的分析结论趋同。

---

## 一、框架的确定性保证

**框架推理是100%确定性的：**

| 环节 | 确定性 | 说明 |
|------|--------|------|
| 预设检测 | ✅ 100% | 内置映射表固定 + 多轮API查询顺序固定 |
| 五态拐点判定 | ✅ 100% | 纯规则匹配，无随机性 |
| 生命周期判定 | ✅ 100% | 纯指标阈值匹配 |
| 个股类型判定 | ✅ 100% | 决策树，无模糊判断 |
| HTML生成 | ✅ 100% | 模板渲染，无随机元素 |

**已清理的遗留代码：**
- V4.3版本的 `import random` 已删除，确保无任何随机性残留

## 二、数据差异的来源（人为因素）

虽然框架是确定性的，但**输入数据可能不同**，原因：

| 差异来源 | 例子 | 影响 |
|---------|------|------|
| 财报时间不同 | 有人搜到Q1，有人搜到Q2 | 营收增速数字不同 |
| 来源优先级不同 | 有人优先采信研报，有人优先采信财报 | 同一指标数字可能不同 |
| 定性→定量转换 | "订单充足" vs "$100M+" | 信号强度判定可能不同 |
| 行业研报版本 | 高盛3月报告 vs 5月更新版 | 行业需求增速数字不同 |
| 汇率/单位换算 | 有人用美元，有人换算成人民币 | 数值不一致 |
| 是否包含子公司 | 合并报表 vs 母公司口径 | 营收数字不同 |

## 三、数据标准化规则

### 3.1 财报数据（最高优先级）

| 字段 | 标准化规则 | 示例 |
|------|-----------|------|
| 营收 | 使用**最新完整季报**（不是快报/预告） | "2026Q1营收1.2亿" |
| 同比变化 | 明确标注对比基准期 | "同比+45.2%（vs 2025Q1）" |
| 毛利率 | 使用**Non-GAAP毛利率**（如有），否则GAAP | "Non-GAAP毛利率25.3%" |
| 订单backlog | 尽量量化，无法量化时标注"定性" | "$100M+" 或 "管理层称'充足'（定性）" |
| 产能利用率 | 优先使用公司披露数字 | "产能利用率85%" |

**来源优先级：**
```
公司财报（一手） > 业绩会纪要（一手口述） > 券商研报（二手） > 行业新闻（三手）
```

### 3.2 行业数据（次高优先级）

| 指标 | 推荐来源 | 标准化格式 |
|------|---------|-----------|
| 市场规模 | LightCounting / Yole / Omdia | "$19.37B（+52% YoY）" |
| 供需缺口 | 行业研报 + 交叉验证 | "缺口70%+（Yole 2026.03）" |
| 产品价格 | 行业新闻 + 多家交叉 | "$2500/片（急单$3000+，2026.02）" |
| 产能利用率 | 行业协会 / 头部企业调研 | "行业整体80%+" |
| 政策数量 | 国务院/工信部/发改委文件 | "3项（2026.01-2026.05）" |

### 3.3 定性描述的标准化

**禁止模糊的定性描述，尽量量化：**

| ❌ 模糊描述 | ✅ 量化描述 |
|-----------|-----------|
| "订单充足" | "订单backlog $100M+（历史新高）" |
| "毛利率提升" | "毛利率从18%→25%（+7pct）" |
| "产能紧张" | "产能利用率85%，扩产50%预计2026Q3投产" |
| "行业景气" | "行业需求增速+52%（LightCounting 2026.03）" |
| "政策利好" | "2项政策：工信部AI算力基建规划 + 发改委数据中心节能标准" |

**无法量化时的标注格式：**
```json
{
  "value": "充足（定性）",
  "source": "CEO业绩会 2026-04-30",
  "note": "管理层未披露具体数字，仅口头表示'订单饱满'"
}
```

### 3.4 时间戳标准化

| 数据类型 | 时间格式 | 示例 |
|---------|---------|------|
| 财报数据 | `YYYY-MM-DD`（财报发布日） | `"2026-04-28"` |
| 行业研报 | `YYYY-MM`（研报月份） | `"2026-03"` |
| 行业新闻 | `YYYY-MM-DD`（新闻日期） | `"2026-02-15"` |
| 政策文件 | `YYYY-MM-DD`（发布日期） | `"2026-01-20"` |

### 3.5 单位标准化

| 指标 | 推荐单位 | 说明 |
|------|---------|------|
| 营收 | 亿元人民币（A股）/ 亿美元（美股） | 标注币种 |
| 毛利率 | 百分比（%） | 明确GAAP/Non-GAAP |
| 订单backlog | 美元或人民币，标注币种 | 与营收单位一致 |
| 产品价格 | 美元（国际定价商品） | 如InP衬底$/片 |
| 产能 | 万片/年、MW、GWh等 | 标注时间单位 |

## 四、数据冲突处理规则

当不同来源给出矛盾数字时：

| 冲突场景 | 处理规则 | 填入方式 |
|---------|---------|---------|
| 财报 vs 研报数字不同 | **优先采信财报**（一手来源） | 用财报数字，标注"研报预测XX，实际YY" |
| 季报 vs 年报调整 | **优先采信最新完整季报** | 用Q1数据，标注"年报可能调整" |
| 公司指引 vs 实际 | **优先采信实际数字** | 用实际数字，标注"公司指引XX，实际YY" |
| 国内 vs 海外来源 | **A股优先国内来源，美股优先海外来源** | 标注来源地域 |
| 时间差导致数字不同 | **优先采信最新数据** | 标注数据截至日期 |

## 五、验证数据一致性

填入数据后，运行验证脚本：

```bash
python scripts/validate_data.py <stock_code>
```

验证项：
- [ ] 所有必填字段已填（不是"待补充"）
- [ ] 每个数字都有 `source` 和 `date`
- [ ] 定性描述已尽量量化
- [ ] 时间戳格式统一（YYYY-MM-DD）
- [ ] 单位统一（币种、数量级）
- [ ] 没有明显的数据矛盾（如营收+50%但毛利率-20%无解释）

## 七、减少差异的最佳实践

1. **财报数据以公司最新季报为准** — 不采信券商预测
2. **行业数据以头部机构为准** — LightCounting、Yole、Omdia、高盛、摩根士丹利
3. **定性描述尽量量化** — "充足"→"$100M+"，"提升"→"从X%→Y%"
4. **时间戳精确到日** — 不写"近期"，写"2026-04-28"
5. **来源写全称** — "公司公告"→"某光芯片公司2026Q1财报（公告编号2026-028）"
6. **矛盾数据标注** — 如果两个来源不同，标注优先采信的理由

## 八、核心结论

| 问题 | 答案 |
|------|------|
| 框架推理是否确定性？ | ✅ **100%确定性**。同样的JSON→同样的HTML |
| 不同的人能否跑出一致结论？ | **大概率相似，核心指标趋同**。取决于数据搜索标准化程度 |
| 差异主要来自哪里？ | **数据获取环节**（不同人搜到不同的数字），不是框架推理环节 |
| 如何最大化一致性？ | 按本指南标准化数据搜索和填入 |

> **框架确保：推理一致。你确保：数据一致。**
