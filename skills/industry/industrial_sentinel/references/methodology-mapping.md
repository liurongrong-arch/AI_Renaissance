# Industrial Sentinel V4.5 — 数据→判断映射规则

> **设计原则**：框架只负责推理逻辑，不负责数据获取。你填入什么数据，框架就按以下规则推理出什么结论。所有规则透明、可审计、可追溯。
>
> **输入**：符合 `data-requirements.md` 格式的 JSON 数据
> **输出**：产业链景气度判定 / 拐点状态 / 生命周期阶段 / 个股类型

---

## 一、System A — 五态拐点判定规则

### 输入信号（来自 JSON `real_signals`）

| 信号 | JSON字段 | 阈值定义 |
|------|---------|---------|
| 营收增速 | `revenue_growth` | >20% = 强信号，0-20% = 弱信号，<0% = 负面 |
| 毛利率 | `gross_margin` | >25% = 强，15-25% = 中，<15% = 弱 |
| 订单backlog | `order_backlog` | 有明确增长数字 = 强，"饱满"/"充足" = 中，未提及 = 弱 |
| 产能利用率 | `capacity_utilization` | >80% = 强，60-80% = 中，<60% = 弱 |
| 产品价格 | `price_yoy` | 涨价 = 强，持平 = 中，跌价 = 弱 |
| 库存天数 | `inventory_days` | 下降 = 强，持平 = 中，上升 = 弱 |
| 扩产计划 | `capex_plan` | 明确扩产 = 强，规划中 = 中，无计划 = 弱 |
| 政策数量 | `policy_count` | >3项 = 强，1-3项 = 中，0项 = 弱 |
| 亏损收窄 | `net_loss_yoy_improvement` | 收窄>50% = 强，收窄 = 中，扩大 = 弱 |

### 推理规则（框架自动执行）

**Step 1：统计强/中/弱信号数量**
```
强信号 ≥ 3 个 → 高置信度
强信号 1-2 个 + 中信号 ≥ 2 个 → 中置信度
强信号 0 个 + 中信号 ≤ 2 个 → 低置信度
```

**Step 2：匹配五态模型**

| 拐点状态 | 信号特征 | 典型组合 |
|---------|---------|---------|
| **拐点前** | 负面信号主导，但出现1-2个改善迹象 | 营收仍跌 + 毛利率触底 + 政策开始出台 |
| **拐点初期** | 改善信号开始增多，但未形成趋势 | 营收转正 + 订单增长 + 毛利率仍低 |
| **拐点确认** | 强信号 ≥3个，形成一致改善趋势 | 营收↑ + 毛利率↑ + 订单↑ + 产能↑ |
| **拐点晚期** | 强信号持续，但出现过热迹象 | 营收高增 + 毛利率高位 + 产能满载 + 价格暴涨 |
| **后衰退** | 负面信号重新主导 | 营收下滑 + 毛利率压缩 + 库存上升 |

**Step 3：输出**
```json
{
  "state_name": "拐点确认",
  "state_reason": "营收+45%、毛利率修复、订单$100M+，三项强信号一致",
  "matched_signals": ["revenue_growth", "gross_margin", "order_backlog"],
  "contradictions": [],
  "confidence": "高"
}
```

### 矛盾处理规则

如果信号之间存在矛盾（如营收↑但毛利率↓）：

| 矛盾组合 | 处理方式 | 输出 |
|---------|---------|------|
| 营收↑ + 毛利率↓ | 优先采信营收，标注"以价换量风险" | "拐点初期（低置信度）" |
| 营收↓ + 订单↑ | 优先采信订单，标注"交付延迟" | "拐点前" |
| 产能↑ + 价格↓ | 标注"供过于求风险" | "拐点晚期（警告）" |
| 政策↑ + 营收↓ | 优先采信政策，标注"政策传导中" | "拐点前" |

## 二、System A — 生命周期判定规则

### 输入指标（来自 JSON `lifecycle_indicators`）

| 指标 | JSON字段 | 判定作用 |
|------|---------|---------|
| 营收增速 | `营收增速` | >50% = 导入期，20-50% = 成长期，<20% = 成熟期 |
| 毛利率修复 | `毛利率修复` | 低→高 = 成长期，高→低 = 成熟期/衰退 |
| 订单backlog | `订单backlog` | 增长中 = 成长期/导入期，稳定 = 成熟期 |
| 产能扩张 | `产能扩张` | 扩张中 = 成长期，满载 = 成熟期，收缩 = 衰退 |
| 行业周期 | `行业周期` | 上行 = 成长期，顶部 = 成熟期，下行 = 衰退 |

### 推理规则

**导入期判定（需同时满足）**
- 营收增速 > 50%（或"高速增长"）
- 毛利率 < 20% 或"亏损"/"低毛利"
- 产能"爬坡中"/"建设中"

**成长期判定（需同时满足）**
- 营收增速 20-50%
- 毛利率"修复中"/"提升"
- 产能"扩张中"/"扩产"

**成熟期判定（需同时满足）**
- 营收增速 < 20%
- 毛利率"稳定"/"高位"
- 产能"满载"/"利用率高位"

**衰退期判定（满足任一）**
- 营收增速 < 0%
- 毛利率"压缩"/"下滑"
- 产能"收缩"/"停产"

### 输出

```json
{
  "stage": "成长期",
  "stage_reason": "营收+45%（成长期区间）、毛利率从18%修复至25%（修复中）、产能扩张50%（扩张中）",
  "key_drivers": ["AI算力需求爆发", "InP衬底紧缺"],
  "risk_factors": ["扩产可能导致产能过剩"]
}
```

## 三、System B — 个股类型判定规则

### 输入指标（来自 JSON `system_b_input`）

| 指标 | JSON字段 | 阈值 |
|------|---------|------|
| 营收增速 | `revenue_growth` | >30% = 高，10-30% = 中，<10% = 低 |
| 研发投入占比 | `rd_ratio` | >5% = 高，2-5% = 中，<2% = 低 |
| 资产模式 | `asset_lightness` | "轻资产" = 高弹性，"重资产" = 低弹性 |
| 盈利稳定性 | `profit_stability` | "稳定盈利" = 价值型，"波动大" = 周期型 |

### 推理规则（决策树）

```
营收增速 > 30%?
  ├─ 是 → 毛利率 > 20%?
  │     ├─ 是 → 研发投入 > 5%?
  │     │     ├─ 是 → **成长型** ✅
  │     │     └─ 否 → 成长型（弱技术壁垒）
  │     └─ 否 → 周期型（高增速但低毛利 = 以价换量）
  └─ 否 → 盈利稳定?
        ├─ 是 → 分红率高?
        │     ├─ 是 → **价值型** ✅
        │     └─ 否 → 价值型（弱分红）
        └─ 否 → 营收与主题强相关?
              ├─ 是 → **主题型** ✅
              └─ 否 → 周期型（营收波动）
```

### 输出

```json
{
  "type": "成长型",
  "type_reason": "营收+45%（>30%）、毛利率25%（>20%）、研发投入8%（>5%），三项均满足成长型阈值",
  "core_contradiction": "需验证：高增速是否来自一次性订单，而非持续性需求",
  "tracking_metrics": ["Q2订单持续性", "新客户认证进度", "产能爬坡进度"],
  "risks": ["估值已反映高增速预期", "扩产后的产能过剩风险"]
}
```

## 四、产业链景气度 — 综合判定规则

### 输入数据

| 维度 | 数据来源 |
|------|---------|
| 行业周期 | `lifecycle_indicators.行业周期` |
| 供需拐点 | `real_signals` 中的价格、产能、订单 |
| 政策催化剂 | `real_signals.policy_count` + `industry_data` 政策项 |

### 推理规则

**景气度上行判定**
- 生命周期 = 成长期/导入期
- 拐点状态 = 拐点初期/拐点确认
- 政策催化剂 ≥ 1项

**景气度顶部判定**
- 生命周期 = 成熟期
- 拐点状态 = 拐点晚期
- 价格暴涨 + 产能满载（过热信号）

**景气度下行判定**
- 生命周期 = 衰退期
- 拐点状态 = 后衰退/拐点前（负面）
- 库存上升 + 价格下跌

### 输出格式

```json
{
  "industry_cycle": "上行 — AI算力驱动光通信进入超级周期",
  "supply_demand": "紧缺 — InP衬底缺口70%+，EML缺口60%+",
  "policy": "强催化 — 十五五AI算力基建列为国家战略",
  "overall": "景气度上行，处于成长期拐点确认阶段"
}
```

## 五、矛盾发现与处理规则

框架在推理过程中会自动发现数据矛盾：

| 矛盾类型 | 发现方式 | 处理规则 |
|---------|---------|---------|
| 信号矛盾 | 营收↑ vs 毛利率↓ | 标注风险，降低置信度 |
| 时间矛盾 | Q1数据 vs Q4数据趋势相反 | 优先采信最新数据 |
| 来源矛盾 | 财报 vs 研报数据不一致 | 优先采信财报（一手） |
| 逻辑矛盾 | 产能扩张 vs 价格暴涨 | 标注"供不应求"，提升景气度 |
| 预设矛盾 | 标的行业与preset不匹配 | 提示用户检查 |

### 矛盾输出

```json
{
  "contradictions": [
    {
      "type": "信号矛盾",
      "description": "营收+45%但毛利率从30%降至25%",
      "resolution": "以价换量可能性，降低拐点置信度至中",
      "priority": "高"
    }
  ]
}
```

## 六、数据缺失降级规则

如果某些必填数据缺失，框架不会报错，而是降级输出：

| 缺失数据 | 降级方式 | 输出变化 |
|---------|---------|---------|
| 所有五态信号缺失 | 无法判定拐点 | "拐点状态：数据缺失 — 请补充至少3项信号" |
| 部分信号缺失 | 基于已有信号判定，降低置信度 | 置信度从"高"降至"中"或"低" |
| 行业数据缺失 | 产业链卡片正常显示，数据表格留空 | "数据缺失"占位 |
| System B指标缺失 | 无法判定个股类型 | "个股类型：数据缺失" |

## 七、完整推理链路图

```
输入：JSON数据（符合 data-requirements.md 格式）
  │
  ├─→ System A 五态信号分析
  │     ├─ 统计强/中/弱信号
  │     ├─ 匹配五态模型
  │     ├─ 发现矛盾（如有）
  │     └─ 输出：拐点状态 + 置信度
  │
  ├─→ System A 生命周期分析
  │     ├─ 匹配生命周期指标
  │     ├─ 判定阶段（导入/成长/成熟/衰退）
  │     └─ 输出：生命周期阶段 + 关键驱动因素
  │
  ├─→ 产业链景气度综合
  │     ├─ 整合生命周期 + 拐点 + 政策
  │     └─ 输出：景气度判定（上行/顶部/下行）
  │
  ├─→ System B 个股分析
  │     ├─ 判定个股类型（成长/周期/价值/主题/混合）
  │     ├─ 识别核心矛盾
  │     ├─ 提出跟踪指标
  │     └─ 输出：个股类型 + 风险清单
  │
  └─→ HTML报告生成
        ├─ 产业链结构卡片（从YAML加载，与数据无关）
        ├─ 数据监测表格（展示填入的数据）
        ├─ 分析结论（基于推理规则自动生成）
        └─ 风险提示（基于矛盾发现）
```

---

## 八、为什么这是方法论而非黑箱

1. **规则透明**：所有推理规则都在本文档中明示，你可以审计每一条逻辑
2. **输入可控**：你决定填入什么数据，框架只负责按规则推理
3. **输出可追溯**：每个结论都标注了基于哪些数据、用了哪条规则
4. **可自定义**：你可以修改 YAML 中的阈值、添加新的信号、调整权重
5. **无隐藏逻辑**：没有神经网络、没有黑箱模型、没有不可解释的评分

## 九、与其他AI/Agent的协作方式

你的Agent只需要做两件事：

**第一步：搜索数据**
```python
# 你的Agent搜索财报、研报、新闻
revenue = search("688XXX.SH Q1 2026 营收")  # → 1.2亿
gross_margin = search("688XXX.SH 毛利率")     # → 25%
order_backlog = search("某光芯片公司 订单")      # → $100M+
```

**第二步：填入JSON**
```python
data = {
    "real_signals": {
        "revenue_growth": 45.2,
        "revenue_growth_source": "2026Q1财报",
        "gross_margin": 25.0,
        "gross_margin_source": "2026Q1财报",
        "order_backlog": "$100M+",
        "order_backlog_source": "CEO业绩会 2026-04-30"
    }
}
# 保存到 data/688XXX.SH_real_data.json
```

**第三步：框架自动推理**
```bash
./run.sh 688XXX.SH
# 框架自动：
# - 统计信号 → 3个强信号 → 拐点确认（高置信度）
# - 匹配生命周期 → 营收45%+毛利率修复+产能扩张 → 成长期
# - System B判定 → 营收>30%+毛利率>20%+研发>5% → 成长型
# - 生成HTML报告
```

**你的Agent不需要理解推理规则**，只需要按要求格式填入数据。框架的推理逻辑是固定的、透明的、可审计的。
