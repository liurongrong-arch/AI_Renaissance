# 财报分析技能 — 用现金验证利润

## 核心思想

**新手看利润表，高手看现金流量表和资产负债表。**

利润可以调节（财务手段合法），但现金的流动和资产的堆积往往更难造假。本技能的核心是用现金流量表和资产负债表的数据，去验证利润表的真实性和可持续性。

## 数据溯源规则

> 没有来源的数据，容易造成误判。关键数据需要标注出处，方便后续复核。

### 规则1：关键数据标注来源

报告中的**定量数据**（金额、比率、同比变化等）建议标注数据来源，格式：

```
[数据值] ← [来源标识]
```

**来源标识规范：**

| 来源 | 标识格式 | 示例 |
|------|---------|------|
| 东方财富API（交易所公告原文） | `东财API·<三表类型>·<报告期>` | `东财API·资产负债表·2026Q1` |
| westock-data | `westock·<命令>·<股票代码>` | `westock·finance·sz300757` |
| neodata-financial-search | `neodata·<查询关键词>` | `neodata·营收净利润` |
| 巨潮资讯网PDF公告 | `巨潮PDF·<公告标题>·<页码>` | `巨潮PDF·2026年一季报·P12` |
| 上市公司官方公告/新闻稿 | `公司公告·<日期>·<标题>` | `公司公告·20260325·新签订单6亿` |
| 二手财经媒体 | `⚠️<媒体名>·<日期>` | `⚠️新浪财经·20260429` |

### 规则2：来源可信度分层

**高可信（可直接引用，无需二次验证）：**
- 东方财富API（数据源 = 巨潮资讯网公告原文）
- 巨潮资讯网PDF公告原文
- 上市公司官方公告

**中可信（可用，但需注明是二手来源）：**
- westock-data（来自腾讯自选股，与公告可能有1-2天延迟）
- neodata-financial-search

**低可信（仅作参考，需要加⚠️标记）：**
- 财经媒体报道（新浪、雪球、同花顺等）
- 研报数据（非原始公告）
- 社交媒体/论坛

### 规则3：三类数据的不同处理

**A类：直接从API/公告获取的原始数据** → 正常引用
```
合同负债 1.795亿 ← 东财API·资产负债表·2026Q1
```

**B类：基于原始数据计算得出的比率/估算** → 标注"计算"并写出公式
```
净债务率 67.0% ← 计算：(短借9.923+长借3.361+一年内到期0.865-货币资金3.253)/股东权益16.256 ← 东财API·资产负债表·2026Q1
收现比 1.77 ← 计算：销售收现2.901亿/营收1.638亿 ← 东财API·现金流量表+利润表·2026Q1
```

**C类：无法从任何官方渠道验证的数据** → 需要标红⚠️，不作为核心判断依据
```
⚠️ 新签订单45.16亿 ← 公司公告·20260428·未经审计
⚠️ 产能利用率 约80% ← ⚠️券商研报·无法验证
```

### 规则4：报告末尾附数据源清单

每份分析报告末尾建议列出本报告使用的数据源及其可信度，方便读者交叉验证：

```
## 数据源清单
| 数据源 | 可信度 | 覆盖范围 |
|--------|-------|---------|
| 东方财富API（巨潮资讯网） | ★★★★★ | 资产负债表+利润表+现金流量表 |
| westock-data | ★★★★ | 历史多期财务数据 |
| 公司公告 | ★★★★★ | 新签订单、业务进展 |
| 新浪财经 | ⚠️★★ | 费用拆分细节 |
```

## 分析框架：七步验证链

整个逻辑链是：**订单真实 → 现金先回 → 应收没恶化 → 存货和在建工程抬升 → 资本开支放量 → 继续扩产兑现**。这是一个从订单到产能释放的完整闭环。

### 第一步：看现金 — 利润是不是被现金撑住

**核心比率：**
- **经营现金流/归母净利润**：长期>1为优，说明每1块钱利润有超1块现金支撑；长期<1甚至为负，利润质量存疑
- **销售收现/营收**：接近1为理想，说明销售基本收回现金；远低于1则大量收入挂在应收上，坏账风险大

**关键检查：**
- 利润增长但现金没跟上 → 纸面富贵，需深挖原因
- 应收账款变化：跟着收入一起恶化，还是在收入增长时反而下降
- 应收下降+收现增强 → 需求真实，公司议价能力强

### 第二步：看需求 — 营运资本位置对不对

**积极信号组合：**
- 应收下降 + 收现增强 → 需求真实，客户愿意提前付钱
- 存货上升 + 在建工程上升 → 不是盲目生产，而是为未来交付做准备

**消极信号组合：**
- 应收上升 + 收现下降 → 需求虚胖，客户拖欠
- 存货上升 + 在建工程下降 → 可能是滞销积压

### 第三步：看业绩 — 合同负债/预收款是先行指标

> **注意区分**：这里说的是**预收款**（客户预付给公司的钱，负债科目），不是预付款（公司付给供应商的钱，资产科目）。

**关键指标：**
- 合同负债（预收款项/预收款）上升 → 订单已开始变成现金和生产动作，不是空头支票
- 预收款是业绩的先行指标，比分析师预测都靠谱

**判断逻辑：**
- 合同负债同比大增 → 下一周期业绩有保障
- 合同负债同比下滑 → 未来收入可能承压

### 第四步：看产能 — 是否已在扩产

**三指标联动：**
1. 在建工程增长
2. 固定资产增长
3. 购建固定资产支付的现金增长

三指标同时增长 → 公司真金白银在投未来，不是光喊口号。

### 第五步：看扩张 — 资本开支力度

**核心数据：** 购建固定资产支付的现金（现金流量表）
- 比利润表里的折旧摊销更能反映当下投资力度
- 资本开支放量 → 往往预示新一轮增长周期开始
- 资本开支缩量 → 公司可能进入收缩期或维持期

### 第六步：看扩张风险 — 净债务水平

**重点分析：**
- 短期借款变化趋势
- 长期借款变化趋势
- 财务费用是否开始明显侵蚀利润

**危险信号：**
- 短期借款暴增 → 可能资金链紧张
- 长期借款大增 + 资本开支未放量 → 借钱不是为了扩产，要警惕
- 财务费用增速 > 营收增速 → 杠杆开始反噬

### 第七步：看利率敏感度 — 财务费用侵蚀

**在利率上行周期尤其警惕：**
- 财务费用/营业利润 > 20% → 高风险
- 财务费用/营业利润 10%-20% → 需密切监控
- 财务费用/营业利润 < 10% → 相对安全

很多公司倒在盲目扩张后，被利息支出拖垮。净债务的规模和结构需要放在显微镜下看。

## 执行流程

### 1. 获取财务数据

**数据源优先级（从权威到补充）：**

**第一优先：东方财富API（来自交易所公告原文，最权威最完整）**

通过 web_fetch 调用东方财富的财务数据接口，获取结构化财报数据。这些数据来自巨潮资讯网公告原文，是最权威的数据源。

```
资产负债表：https://emweb.eastmoney.com/NewFinanceAnalysis/zcfzbAjaxNew?companyType=4&reportDateType=0&reportType=1&dates=2026-03-31&code=SZ300757
利润表：https://emweb.eastmoney.com/NewFinanceAnalysis/lrbAjaxNew?companyType=4&reportDateType=0&reportType=1&dates=2026-03-31&code=SZ300757
现金流量表：https://emweb.eastmoney.com/NewFinanceAnalysis/xjllbAjaxNew?companyType=4&reportDateType=0&reportType=1&dates=2026-03-31&code=SZ300757
```

- 修改 `dates` 参数获取不同报告期（格式：YYYY-MM-DD）
- 修改 `code` 参数适配不同股票（格式：SH600519 / SZ300757）
- 优势：数据最完整、最权威，包含所有科目（包括合同负债、短期借款等westock-data可能遗漏的科目）

**第二优先：westock-data（适合快速获取历史多期数据）**

```bash
# 搜索确认股票代码
npx --yes westock-data-skillhub@latest search <公司名>

# 获取财务报表（最近4期）
npx --yes westock-data-skillhub@latest finance <股票代码> 4
```

代码格式：沪市 sh600519 / 深市 sz000001 / 港股 hk00700 / 美股 usAAPL

注意：westock-data 对最新季报数据更新可能有1-2天延迟，刚披露的财报建议优先用东方财富API。

**第三优先：neodata-financial-search**

当上述两个数据源均不可用时，补充使用 `neodata-financial-search` skill 查询特定指标。

### 2. 提取关键数据

从财务报表中提取以下核心数据（至少最近4个报告期）：

**利润表：**
- 营业收入
- 归母净利润

**现金流量表：**
- 经营活动现金流净额
- 销售商品提供劳务收到的现金
- 购建固定资产无形资产支付的现金

**资产负债表：**
- 应收账款
- 存货
- 合同负债（预收款项）
- 在建工程
- 固定资产
- 短期借款
- 长期借款
- 财务费用

### 3. 计算核心比率

按照 `references/财报指标详解.md` 中的公式，逐项计算七个维度的核心比率和趋势。

按照以下结构输出分析报告（**定量数据需要标注来源**）：

```

> 报告期：<YYYYQX>  |  分析时间：<日期>
> 数据来源：<列出主要数据源及可信度>

### 一、利润现金验证
- 经营现金流/归母净利润：[数值] ← [来源] → [判断]
- 销售收现/营收：[数值] ← [来源] → [判断]
- 利润质量总评：[高/中/低]

### 二、需求真实性验证
- 应收账款变化：[数值] ← [来源] | [趋势]
- 收现能力变化：[数值] ← [来源] | [趋势]
- 营运资本信号：[积极/消极/中性]

### 三、业绩先行指标
- 合同负债：[数值] ← [来源] | 同比[变化率] ← [来源] → [判断]

### 四、产能扩张信号
- 在建工程：[数值] ← [来源] | [趋势]
- 固定资产：[数值] ← [来源] | [趋势]
```

### 5. 辅助工具

对于需要批量计算的场景，使用 `scripts/analyze_report.py` 脚本自动化数据提取和比率计算。详细用法见脚本内文档。

## 注意事项

- 至少获取最近4个报告期数据，才能判断趋势
- 港股/美股注意货币单位标注（港股为港币，美股为美元）
- 季度数据有季节性波动，建议同比而非环比
- 银行、保险等金融行业不适用本框架（资产负债结构完全不同）
- 单一指标异常不构成结论，需多维度交叉验证
