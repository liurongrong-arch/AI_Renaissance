# 量价背离与反转信号识别

> 理论来源：Goldman Sachs QIS STR 因子家族，结合 Campbell-Grossman-Wang (1993)、Lee-Swaminathan (2000)、Gervais-Kaniel-Mingelgrin (2001) 等学术经典整理。

---

## 1. 适用范围

所属小组：专家2组（指标）

适用任务：
- 个股、板块、指数的短期量价背离识别（顶背离 / 底背离 / 量价齐升后乏力 / 量价齐跌后衰竭）
- 事件驱动后的反转概率判断（财报公告、研报调评、指数调仓）
- 持仓股短期超涨减仓时机 / 目标股低位建仓时机判断
- 适合时间周期：5~21 个交易日（短期），A 股建议优先取 5~10 日

### 边界说明

- 强趋势牛市（过去 3 个月指数涨幅 > 20% 且仍加速）中，动量主导，反转策略失效，输出 `neutral`
- 极端流动性枯竭阶段（市场出现系统性危机、全面踩踏）中，价格冲击不回归，输出 `neutral`
- A 股以下场景不适用，输出 `neutral` 并标记 `needs_human_review: true`：
    
    - 政策强刺激期（如单日指数涨超 3%、全面利好政策出台）
    - 游资题材股 / 妖股（连续涨停且无基本面支撑）
    - ST 股 / 退市风险股（±5% 涨跌停 + 基本面恶化）
    - 北交所股票（流动性差，须严格执行 §4.4 过滤器三 Amihud 和过滤器四绝对成交额双重检查；不达标即输出 `neutral`）
    - 重大节假日前后 2 个交易日（春节 / 国庆资金季节性扰动）
- 成交量数据缺失或历史数据不足 60 日时，需降低 `confidence` 并标记人工复核

## 2. 输入材料

### 必填输入

- 标的：股票代码 / 板块名称 / 指数代码
- 时间范围：分析窗口（至少包含过去 60 个交易日行情数据）
- 核心行情数据：
  - 日线收盘价序列（用于计算 STR 和形态判断）
  - 日线最高价 / 最低价（用于识别上影线 / 下影线形态）
  - 日线成交量序列（用于计算 VolSurp 和均量比较）
  - 换手率序列（用于换手率分层，至少 20 日）
- 数据来源：行情数据（market_data），来自东方财富 / 同花顺 / Wind 等

### 可选输入

- RSI 指标数值（RSI6 / RSI12 / RSI24），用于复合判断规则
- 主力资金净流入数据（超大单净额），来自资金流数据（fund_flow）
- A 股涨停板信息（是否涨停、封板时间、打开次数）
- A 股集合竞价价格（9:25 竞价价格 vs 昨收比较）
- 融资余额变化（连续 N 日净增 / 净减）
- 板块 / 行业成交量均值（用于板块超涨超跌判断）

### 缺失处理

- 缺少成交量数据 → 无法执行任何量价判断，输出 `direction: "neutral"`，`confidence` 降至 0.2，`meta.needs_human_review: true`，`meta.uncertainties` 写明「缺少成交量数据，无法完成量价分析」
- 缺少换手率数据 → 跳过换手率分层，`confidence` 降低 0.1，在 `meta.uncertainties` 说明「换手率分层未执行，信号权重未调整」
- 历史数据不足 60 日 → `confidence` 降至 0.3，`meta.needs_human_review: true`
- 缺少 RSI 数据 → 仅执行量价形态判断，跳过 RSI 复合判断规则；`meta.uncertainties` 写明「RSI 缺失，复合信号不完整」
- 可选输入缺失 → 继续分析，但在 `meta.uncertainties` 中说明对信号覆盖度的影响

## 3. 分析步骤

1. **确认分析对象与市场环境**：检查标的所属市场（A 股主板 / 科创板 / 创业板 / 北交所 / 港股 / 美股），判断是否处于强趋势牛市或极端流动性枯竭，若是则直接输出 `neutral`。
2. **检查数据完整性**：核对必填输入是否齐全；缺失时按缺失处理规则降低 `confidence` 或输出 `neutral`。
3. **计算量价核心指标**：
   - 计算 STR（横截面短期反转得分）
   - 计算 VolSurp（异常成交量 Z 分数）
   - 计算换手率分位，确定分层系数
   - 计算 Amihud ILLIQ，判断流动性是否可执行
   - **计算绝对成交额**（当日 + 过去 20 日均值），对照 §4.4 过滤器四阈值表，确定是降权 ×0.5 还是强制 neutral
4. **A 股涨跌停前置过滤**（若标的为 A 股）：检查当日是否涨停一字板/跌停一字板/未开板封死，若是则跳过形态一/二的顶/底背离判断，转入 §4.7 涨停板规则。
5. **识别量价形态**：依据第 4 节判断规则，匹配以下四种形态中的一种或多种：顶背离 / 底背离 / 量价齐升后乏力 / 量价齐跌后衰竭；**形态三/四须按 §4.2 长上下影线精确定义（三条件全成立）判定**，并区分是否为十字星犹豫形态。
6. **执行 RSI + 空间位置三维度复合验证**：按 §4.3 复合判断三维度（量价 + 动量 + 空间）规则，校验信号是否撞到关键压力位/支撑位；空间未确认的反转信号显著降权并写入 `meta.uncertainties`。
7. **执行 A 股特有规则**（若标的为 A 股）：按第 4.7 节处理涨停板特有形态、T+1 执行约束、参数阈值调整。
8. **确定 direction / confidence / risk_level**：汇总所有信号，按第 4 节规则映射，对照 §4.4 指标解读速查表逐项累加 confidence 调整量。
9. **整理 evidence、key_findings 和命中逻辑说明**：
   - 记录触发规则的指标值、数据来源、对比基准；
   - **对每个被命中"强制约束等级"的指标**（如 STR 中度/强烈、VolSurp 爆量/巨量/缩量/地量、换手率高/极端/低、影线标准/极端、流动性差、绝对成交额不足/长期不足等），**必须**从 §4.4 速查表的「命中逻辑说明」列**原文引用**对应一句话，写入：
     - 标准 JSON 的 `reasoning` 字段（多指标命中时按"指标名 → 命中逻辑"句式拼接为一段）
     - 标准 JSON 的 `meta.evidence[].note` 字段（每条 evidence 对应一个指标，note 写该指标命中等级 + 命中逻辑说明）
     - §5.2 Markdown 摘要中该指标行的「**→ 意味着**：……」子项
   - 命中"平淡/正常/中等/容量充足/非长影线"等中性等级时**不需要写**命中逻辑说明，避免输出冗余。
10. **输出结果**：
   - **Agent 系统调用**：按第 5 节格式输出标准 JSON，供 `Signal.from_dict()` 解析；
   - **人工 / Cursor 直接调用**：按第 5.2 节格式输出 Markdown 摘要，省略 JSON，提升可读性。
   - 不确定调用场景时，**优先输出 JSON**。

## 4. 判断规则

### 4.1 量价配合基础过滤

在执行任何形态判断之前，先通过成交量区分信号类型：

| 价格信号 | 成交量信号 | 判断 | direction 倾向 |
|---|---|---|---|
| 大涨 / 大跌 | 异常放大（VolSurp > +2，约 > 均值 2 倍） | 流动性冲击或情绪交易 | 反转概率高，进入形态判断 |
| 大涨 / 大跌 | 正常或萎缩 + 价格创新高/低 | 可能是知情交易（基本面信息） | 反转概率低，倾向 `neutral` 或动量延续 |
| 小涨 / 小跌 | 异常放大（VolSurp > +2） | 多空对峙，筹码换手 | 方向待确认，结合其他规则判断 |
| **涨停 / 跌停且未开板** | 缩量（封板锁筹/惜售）或巨量 | A 股涨停板/跌停板特有形态，**不按顶/底背离处理** | **跳过形态一/二，转入 §4.7 涨停板规则** |

- 若价格大幅变动而成交量正常（VolSurp 在 -1 ~ +1 之间），`confidence` 降低 0.15，在 `meta.uncertainties` 写明「量价无明显异常，反转信号可信度偏低」
- 涨停/跌停的判定阈值：主板 ±9.7%（≈10%）、科创板/创业板 ±19.7%（≈20%）、北交所 ±29.7%（≈30%）、ST 股 ±4.7%（≈5%）；当日全天封死未开板时，按本表最后一行处理

---

### 4.2 四种核心量价背离形态

#### 形态一：顶背离（Top Divergence）

**前置排除**（满足任一即跳过本形态，转入对应规则）：
- 当日**涨停一字板**或**涨停未开板**（A 股缩量新高的特殊情形）→ 主力锁仓惜售，属强势 bullish 而非顶背离，转入 §4.7「缩量全天封死」规则
- 当日**集合竞价开盘即涨停**且全天无开板 → 同上，转入 §4.7

> 说明：一字板/缩量涨停同样满足"创新高 + 成交量萎缩"，但本质是**主力锁仓**而非动能衰竭，绝不可判为 bearish 顶背离。本前置排除是 A 股语境下避免方向错判的硬约束。

**触发条件**（前置排除未命中后）：
- 价格创 N 日新高（N 取 20 个交易日）
- 成交量较 N 日均值**萎缩**（VolSurp < -1，即成交量 < 20 日均量的 80%）
- **且当日未涨停**（涨幅 < 涨停板阈值的 95%，例如主板 < 9.5%）

**量化对应**：STR 因子打分 -1 ~ -2（看跌）

| 字段 | 值 |
|---|---|
| direction | `bearish` |
| confidence | 0.60 ~ 0.75；若同时 RSI 不创新高，升至 0.75 ~ 0.85 |
| risk_level | `medium` |

---

#### 形态二：底背离（Bottom Divergence）

**前置排除**（满足任一即跳过本形态，转入对应规则）：
- 当日**跌停一字板**或**跌停未开板**（A 股缩量新低的特殊情形）→ 主力出货完毕或恐慌锁定，需观察次日竞价是否打开，转入 §4.7「跌停/炸板后反转」规则
- 当日**集合竞价开盘即跌停**且全天无开板 → 同上，转入 §4.7

> 说明：跌停一字板同样满足"创新低 + 成交量萎缩"，但本质是**流动性缺失下的强制锁定**，反转需等次日竞价或开板后判断，不可直接判为 bullish 底背离。

**触发条件**（前置排除未命中后）：
- 价格创 N 日新低（N 取 20 个交易日）
- 成交量较 N 日均值**萎缩**（VolSurp < -1，即成交量 < 20 日均量的 80%）
- **且当日未跌停**（跌幅 < 跌停板阈值的 95%）

**量化对应**：STR 因子打分 +1 ~ +2（看涨）

| 字段 | 值 |
|---|---|
| direction | `bullish` |
| confidence | 0.60 ~ 0.75；若同时 RSI 不创新低，升至 0.75 ~ 0.85 |
| risk_level | `low` |

---

#### 形态三：量价齐升后乏力（Volume Climax Up）

**触发条件**（同时满足）：
- 单日涨幅 > 1.5σ（σ 为过去 60 日日收益率标准差，A 股建议阈值 > 5%）
- 价格创近期高点
- 成交量 > 5 日均值的 2~3 倍（VolSurp > +2）
- K 线收出**长上影线**（按下方"长上影线精确定义"判定，三条件全部成立）

**长上影线精确定义**（三条件**全部**满足才算长上影线）：

记：上影线 = 最高价 - max(开盘价, 收盘价)，实体 = |收盘价 - 开盘价|，全日振幅 = 最高价 - 最低价

| 条件 | 公式 | 含义 |
|---|---|---|
| **条件 1**：影线显著大于实体 | 上影线 ≥ 实体 × 2 | 排除"中阳带浅上影"等非反转 K 线 |
| **条件 2**：影线占振幅主导 | 上影线 / 全日振幅 ≥ 0.5 | 全天波动有一半以上是被打回的，确认抛压 |
| **条件 3**：全日振幅显著 | 全日振幅 / 昨收 ≥ 3% | 排除窄幅震荡造成的"假上影" |

**confidence 细化**：

| 上影线 / 振幅 | confidence 区间 | 含义 |
|---|:---:|---|
| 50% ~ 70% | 0.75 ~ 0.82 | 标准长上影，反转信号成立 |
| > 70% | 0.82 ~ 0.90 | 流星线/吊颈线级强反转 |

**特殊形态：十字星（Doji）**：若实体 / 振幅 < 0.1（极小实体）且上下影线均较长 → 不判为形态三/四，单独输出「十字星犹豫」形态：`direction: "neutral"`，`confidence: 0.40 ~ 0.50`，`risk_level: "medium"`，写入 meta.key_findings「高位/低位十字星，多空胶着，等待次日方向选择」

**量化对应**：STR 因子打分 -2 ~ -3（强看跌）

| 字段 | 值 |
|---|---|
| direction | `bearish` |
| confidence | 0.75 ~ 0.90（按上影线 / 振幅比例细化） |
| risk_level | `high` |

**A 股制度执行约束（写入 meta.key_findings）**：巨量长上影线信号须在当日尾盘（14:50 前）执行；科创板/创业板可用 15:05~15:30 盘后定价

---

#### 形态四：量价齐跌后衰竭（Volume Climax Down）

**触发条件**（同时满足）：
- 单日跌幅 > 1.5σ（A 股建议阈值 > 5%）
- 价格创近期低点
- 成交量 > 5 日均值的 2~3 倍（VolSurp > +2）
- K 线收出**长下影线**（按下方"长下影线精确定义"判定，三条件全部成立）

**长下影线精确定义**（三条件**全部**满足才算长下影线）：

记：下影线 = min(开盘价, 收盘价) - 最低价，实体 = |收盘价 - 开盘价|，全日振幅 = 最高价 - 最低价

| 条件 | 公式 | 含义 |
|---|---|---|
| **条件 1**：影线显著大于实体 | 下影线 ≥ 实体 × 2 | 排除"中阴带浅下影"等非反转 K 线 |
| **条件 2**：影线占振幅主导 | 下影线 / 全日振幅 ≥ 0.5 | 全天波动有一半以上被买盘拉回，确认承接 |
| **条件 3**：全日振幅显著 | 全日振幅 / 昨收 ≥ 3% | 排除窄幅震荡造成的"假下影" |

**confidence 细化**：

| 下影线 / 振幅 | confidence 区间 | 含义 |
|---|:---:|---|
| 50% ~ 70% | 0.70 ~ 0.78 | 标准长下影，承接信号成立 |
| > 70% | 0.78 ~ 0.85 | 锤子线/倒锤线级强承接 |

**十字星形态**：与形态三共用，参见上节定义。

**量化对应**：STR 因子打分 +2 ~ +3（强看涨）

| 字段 | 值 |
|---|---|
| direction | `bullish` |
| confidence | 0.70 ~ 0.85（按下影线 / 振幅比例细化） |
| risk_level | `medium` |

**补充条件**：若标的近期有重大基本面负面事件（业绩暴雷、诉讼、退市风险），`confidence` 降至 0.4，`risk_level` 升为 `high`，`needs_human_review: true`

**A 股制度执行约束（写入 meta.key_findings）**：买入信号当日可执行，但受 T+1 制度约束次日才能卖出，需预留 1 个交易日缓冲

---

### 4.3 日 K / 周 K 复合判断规则

RSI 与量价配合后信号更可靠；但**纯量价 + RSI** 仍不足以避免「下跌中继中的假底背离」与「上涨中继中的假顶背离」——需引入**空间维度**（支撑位/压力位）作为第三确认。

#### 复合判断三维度

| 维度 | 指标 | 作用 |
|---|---|---|
| **量价** | VolSurp（萎缩/爆量） | 确认动量异常 |
| **动量** | RSI（不创新高/新低） | 确认动能背离 |
| **空间** | 支撑位/压力位附近 | 确认反转发生在「有意义的价格位置」，避免下跌中继 / 上涨中继的假信号 |

#### 空间位置确认（关键参考位）

**底背离的支撑位**（满足任一即视为「在支撑位附近」，价格距离支撑位 ≤ 3%）：
- MA60 日均线（中期趋势分水岭）
- 周线 MA20（中期均线）
- 前期成交密集区下沿（筹码集中区）
- 布林下轨（统计学下边界）
- 斐波那契回撤 0.618 / 0.5 位
- 前期重要低点（双底/三重底的颈线对应低点）

**顶背离的压力位**（满足任一即视为「在压力位附近」，价格距离压力位 ≤ 3%）：
- 前期重要高点
- 周线 MA60（长期趋势线）
- 布林上轨
- 斐波那契回撤 0.618 / 0.5 位（反弹位）
- 前期成交密集区上沿（套牢盘集中区）

#### 三维度复合规则表

| 价格信号 | RSI 信号 | 成交量信号 | 空间确认 | 综合判断 | direction | confidence |
|---|---|---|---|---|---|---|
| 价格创新高 | RSI 也创新高 | 正常 | — | 健康上涨，趋势延续 | `neutral` | 0.30 |
| 价格创新高 | RSI **不创新高** | **萎缩** | ✅ 在压力位附近 | 顶背离 + 空间确认，**三重共振** | `bearish` | 0.85 ~ 0.92 |
| 价格创新高 | RSI **不创新高** | **萎缩** | ❌ 未在压力位 | 顶背离但**可能是上涨中继**，降权 | `bearish` | 0.55 ~ 0.65（risk_level 升 `high`，写入 `meta.uncertainties`「空间压力位未确认，警惕上涨中继」） |

| 价格创新低 | RSI **不创新低** | **萎缩** | ✅ 在支撑位附近 | 底背离 + 空间确认，**三重共振** | `bullish` | 0.85 ~ 0.92 |
| 价格创新低 | RSI **不创新低** | **萎缩** | ❌ 未在支撑位 | 底背离但**可能是下跌中继**，降权 | `bullish` | 0.55 ~ 0.65（risk_level 升 `high`，写入 `meta.uncertainties`「空间支撑未确认，警惕下跌中继」） |
| 价格大涨 5%+ | — | 爆量 + 长上影线 | ✅ 在压力位附近 | 高位放量滞涨 + 撞压力 | `bearish` | 0.82 ~ 0.90 |
| 价格大涨 5%+ | — | 爆量 + 长上影线 | ❌ 未在压力位 | 高位放量滞涨 | `bearish` | 0.70 ~ 0.78 |
| 价格大跌 5%+ | — | 爆量 + 长下影线 | ✅ 在支撑位附近 | 低位放量承接 + 守支撑 | `bullish` | 0.78 ~ 0.85 |
| 价格大跌 5%+ | — | 爆量 + 长下影线 | ❌ 未在支撑位 | 低位放量但位置不佳 | `bullish` | 0.60 ~ 0.70 |

> **降权处理逻辑**：未通过空间确认的反转信号，`direction` 仍保留原方向（不翻转为 neutral），但 `confidence` 显著下调、`risk_level` 升档、并强制写入 `meta.uncertainties`。理由：空间维度是**信号质量过滤器**而非**方向决定器**，量价 + 动量已基本确定方向，空间维度决定的是这次反转是「真反转」还是「中继过程」。

**量级定义**：
- 日 K 爆量：成交量柱 > 5 日均量线 200%
- 日 K 地量：成交量柱 < 5 日均量线 70%
- 周 K 爆量：同样参考 5 周均量；周线爆量 + 新高 + 长上影 + 周线压力位 → `bearish`，confidence 升至 0.88~0.93
- 周 K 地量：5 周均量以下 70%；地量 + 低位横盘 + RSI < 30 + 周线支撑位 → `bullish`，confidence 升至 0.88~0.93

### 4.4 量化公式（精确阈值依据）

#### 主信号：横截面短期反转（STR）

\[
\text{STR}_{i,t} = -\,\text{Z}\big(R_{i,[t-L,\,t-1]}\big)
\]

- \(R_{i,[t-L,\,t-1]}\)：股票 \(i\) 在过去 \(L\) 个交易日的累计收益率
- L 取 5（一周）或 21（一月）；A 股优先取 5~10
- Z 是横截面标准化（剔除行业、市值、Beta 后做残差）
- 取负号：过去涨多者打负分（bearish），过去跌多者打正分（bullish）

> **白话解读**：STR 衡量"**最近一段时间相对同行涨太多/跌太多的程度**"。把过去 L 日累计收益率在全市场（或同板块）做标准化打分，前 5% 涨幅最高者拿到 +2 标准差，再**取负号**变成 −2 ——意思是"涨得太凶，反弹动力衰减，应转跌"。底层逻辑：短期价格冲击会均值回归（mean reversion），而中长期才是动量延续。

#### 量价过滤器一：异常成交量（Volume Surprise）

\[
\text{VolSurp}_{i,t} = \frac{V_{i,t} - \overline{V_{i,[t-20,\,t-1]}}}{\sigma\big(V_{i,[t-60,\,t-1]}\big)}
\]

- VolSurp > +2（放量超过 2 个标准差）：反转信号权重 ×1.5~2 倍，confidence +0.10~0.15
- VolSurp < -1（明显缩量）：结合价格创新高/新低触发顶/底背离规则

> **白话解读**：VolSurp 衡量"**今天成交量是不是异常**"。分子是「今日量 − 近 20 日平均量」，分母是「近 60 日量的波动幅度」。> +2 大约对应"比平时多 2 倍"，< −1 对应"比平时少 30~50%"。**注意 VolSurp 是相对量**，小盘股放大 5 倍可能也才 2000 万元，所以必须配合过滤器四（绝对成交额）使用。

#### 量价过滤器二：换手率分层（Turnover Tier）

| 换手率分位 | 反转因子调整 | confidence 调整 |
|---|---|---|
| 高换手（前 20%）；A 股：单日 > 5% | 信号 ×1.5 | +0.10 |
| 中等（中间 60%） | 信号 ×1.0 | 0 |
| 低换手（后 20%）；A 股：单日 < 1% | 信号 ×0.5 或剔除 | -0.15；建议输出 `neutral` |

> **白话解读**：换手率（=日成交量 / 流通股本）反映"**筹码换手的活跃度**"。高换手意味着今天有大量人换手——既可能是恐慌出货（顶部）也可能是恐慌买入（底部），所以是"反转信号的放大器"。低换手 + 反转信号 = 几乎没人参与，信号缺乏共识，不可信。

#### 量价过滤器三：流动性（Amihud Illiquidity）

\[
\text{ILLIQ}_{i,t} = \frac{1}{20}\sum_{s=t-20}^{t-1}\frac{|R_{i,s}|}{\text{Dvol}_{i,s}}
\]

- ILLIQ 超过样本第 70 百分位（流动性最差的 30%）→ 直接剔除，输出 `neutral`，`meta.uncertainties` 写明「流动性不足，信号无法执行」
- 剩余 70% 样本正常执行，ILLIQ 较高者 `risk_level` 可酌情升一档

> **ILLIQ 与 VolSurp 的区别**：ILLIQ 衡量「**价格对小额资金的敏感度**」（敏感性），VolSurp 衡量「**成交量相对自身均值的偏离**」（相对性）。两者都不直接看「**绝对成交额是否足够**」（容量性），因此需要下方的过滤器四作为补充。

#### 量价过滤器四：绝对成交额阈值（Dollar Volume Floor）

VolSurp 是**相对均值**的 Z 分数，存在小盘股陷阱——小盘股日均成交额本就很小，单日成交额翻倍也可能仍是"散户群体的情绪交易"，而非真正主力介入。需要**绝对成交额**的硬底过滤。

**分市场绝对成交额阈值**：

| 市场板块 | 单日成交额阈值 | 处理方式 |
|---|---|---|
| **沪深主板**（沪市 60xxxx / 深市 000xxx） | < **1 亿元** | VolSurp 信号 ×0.5；confidence -0.10；`meta.uncertainties` 写明「绝对成交额不足，VolSurp 信号可能失真」 |
| **科创板**（688xxx） | < **5000 万元** | 同上 |
| **创业板**（300xxx） | < **5000 万元** | 同上 |
| **北交所**（83/87/92 开头） | < **2000 万元** | 直接剔除信号，输出 `neutral`，`needs_human_review: true` |
| **港股主板** | < **1000 万港元** | VolSurp 信号 ×0.5；confidence -0.10 |
| **港股 GEM/小盘** | < **500 万港元** | 直接剔除信号，输出 `neutral`，`needs_human_review: true` |
| **美股主板**（NYSE/NASDAQ 主流） | < **1000 万美元** | VolSurp 信号 ×0.5；confidence -0.10 |

**判定规则**：
1. 取**信号触发当日**成交额 与 **过去 20 日均成交额** 中的较低者，对比表中阈值
2. 若过去 20 日均成交额本身 < 阈值 × 50%（即长期低于阈值的一半），**强制 `neutral` + `needs_human_review: true`**，不参与任何反转判断
3. 阈值随市场风格漂移可定期复盘调整，写入 `meta.evidence` 时标注阈值版本

> **过滤器三 vs 过滤器四**：ILLIQ 高的股票"对小钱敏感"，绝对成交额低的股票"容量不够"。两者命中其一就应降权或剔除；同时命中应直接 `neutral` 不执行。

#### 复合打分（最终信号）

\[
\text{Signal}_{i,t} = \text{STR}_{i,t} \times f(\text{VolSurp}_{i,t}) \times g(\text{Turnover}_{i,t}) \times \mathbb{1}\{\text{ILLIQ}_{i,t} \le \text{Q}_{70}\} \times h(\text{DollarVol}_{i,t})
\]

其中 \(h(\cdot)\) 为绝对成交额阈值函数（过滤器四）：达标取 1.0，不足取 0.5，严重不足或长期不足取 0（强制 neutral）。

Signal > 0：`bullish`；Signal < 0：`bearish`；Signal ≈ 0 或 ILLIQ / DollarVol 过滤命中 0：`neutral`

### 4.6 板块 / 行业超涨超跌规则

- 反转预期兑现窗口：未来 2~4 周

#### A 股参数校准（覆盖通用阈值）

| 反转观察窗口 L | 5~21 日 | **5~10 日** | A 股情绪反应周期更短，反转通常 5 日内发生 |

| 信号 | 执行时机调整 |
|---|---|
| 巨量长上影线 → 减仓信号 | 必须在**当日尾盘（14:50 前）执行**；科创板/创业板可用 15:05~15:30 盘后定价 |
| 放量大跌后衰竭 → 买入信号 | 当日可买，但次日才能卖，需预留 1 个交易日缓冲，写入 meta.key_findings |

| 集合竞价 9:25 价格 > 昨收 +5% | 当日极易冲高回落，不追高，`direction: "bearish"`（短期），`confidence: 0.65` |

#### A 股涨停板特有形态

**首板涨停**：

| 成交量与封板情况 | 含义 | direction | confidence | risk_level |
|---|---|---|---|---|
| 爆量 + 早盘封板（9:40 前封死） | 主力强势介入 | `bullish`（动量延续） | 0.60 | `medium` |
| 爆量 + 尾盘封板（14:30 后） | 尾盘突袭，次日可能开板 | `neutral` | 0.40 | `high` |
| 巨量 + 封板反复被砸开 | 主力出货，筹码松散 | `bearish` | 0.75 | `high` |
| 缩量全天封死 | 主力锁仓，惜售 | `bullish`（趋势延续） | 0.65 | `low` |

**连板顶部判断**：
- 2~3 连板：动量延续，不适用反转模型，输出 `neutral`
- 5 板以上，某日高开但量能不及前一天 → `direction: "bearish"`，`confidence: 0.70~0.80`，`risk_level: "high"`

**跌停/炸板后反转**：
- 个股跌停 + 次日竞价恢复正常 + 成交量较跌停日萎缩 → `direction: "bullish"`，`confidence: 0.65~0.75`
- 指数连跌 3~5 日 + 大面积跌停 + 跌停数量逐日减少 → `direction: "bullish"`，`confidence: 0.70`，写入 meta.key_findings「恐慌衰竭，接近阶段底部」

---

### 4.8 信号冲突处理

- 量价形态为 `bearish`，但资金流向持续净流入（大单净额正） → 输出 `neutral`，`confidence: 0.35`，`needs_human_review: true`，写入 `meta.uncertainties`「量价信号与资金流向矛盾」
- 顶背离形态触发，但同期有重大利好公告 → 降低 `bearish` 信号权重，`confidence` 降至 0.45，`needs_human_review: true`
- 多个形态同时触发且方向一致 → `confidence` 上浮 0.05~0.10，不超过 0.95

## 5.1 批量输出规范（多标的场景）【扩展节，模板外】

**触发条件**：仅当分析标的数量 **N ≥ 2** 时输出本节定义的「汇总速览表」。
**单只标的（N=1）不输出速览表**，直接进入 §5.2 摘要即可（首行即核心结论，单只时再加表纯属冗余）。

### 设计意图：形态判断速览（Pattern-Oriented）

速览表的唯一作用是让人**"一眼扫完每个标的的形态判断"**，仅承载三类信号层信息：
① 标的识别　② direction / confidence / risk_level（模板标准字段）　③ 触发的形态名（或未触发原因）

**所有其他信息一律下放到 §5.2 每只标的的详细摘要**：
- 过程指标（STR / VolSurp / RSI / BOLL / 涨幅 / 换手率 / 创新高 / 影线占比 / 最新价 等）→ 放 §5.2 各指标行
- 关键价位、监测触发条件 → 放 §5.2「执行约束与监测触发条件」块
- 仓位指令（减仓 / 加仓 / 清仓 等）→ **不输出**（依据 docs/SKILL_TEMPLATE.md 第 12、82~84 行，Skill 不替仲裁层做仓位决策）

### 表格格式（4 列固定，不得扩展）

```markdown
## 汇总速览表

| 标的（代码） | 方向 / 置信 / 风险 | 触发形态 | 信号说明（≤25字） |
|---|:---:|:---:|---|
| XX XXXXXX | 🔴 bearish / 0.XX / high | 形态X·名称 | 1~2 个核心证据 |
| YY YYYYYY | ⚪ neutral / 0.XX / medium | 无形态触发 | 未触发原因 |
```

### 列填写规则

| 列 | 来源 | 说明 |
|---|---|---|
| **标的（代码）** | 输入 | 公司简称 + 6 位代码同列、空格分隔（如 `立讯精密 002475`），便于一眼识别 |
| **方向 / 置信 / 风险** | JSON `direction` / `confidence` / `meta.risk_level` | emoji（🔴 bearish / ⚪ neutral / 🟢 bullish）+ direction + 两位小数 confidence + risk_level，用 `/` 分隔；如 `🔴 bearish / 0.68 / high` |
| **触发形态** | JSON `signals[0]` 形态字段 | 使用**标准化形态名**：`形态一·顶背离` / `形态二·底背离` / `形态三·量价齐升乏力` / `形态四·量价齐跌衰竭` / `隐性顶背离` / `隐性底背离` / `无形态触发`；若同时命中多形态，写主形态名 |
| **信号说明（≤25字）** | `reasoning` 首句或 `signals[0]` 核心证据 | **限 25 字以内**，仅含 1~2 个核心证据（如"创新高+量未跟+RSI86.58"、"20日+102%透支+换手19%"、"光头光脚多头延续"）；不重复形态名；**禁止出现"减仓/加仓/清仓"等仓位指令及具体监测价位** |

### 输出顺序

| 场景 | 输出顺序 |
|---|---|
| **单标的（N=1），人工调用** | **直接 §5.2 摘要**（跳过速览表） |
| 多标的（N≥2），人工调用 | 汇总速览表 → 标的① §5.2 摘要 → 标的② §5.2 摘要 → … |
| 多标的（N≥2），Agent 系统调用 | 汇总速览表（人工阅读层）→ 各标的 JSON |

汇总表为人工阅读层，不影响 JSON 的机器解析；JSON 仍是标准结构化输出，两者并列存在。

### 自查清单（输出前必查）

1. **列数 = 4**：标的（代码）/ 方向·置信·风险 / 触发形态 / 信号说明——不多不少
2. **触发形态使用标准化名称**：形态一·顶背离 / 形态二·底背离 / 形态三·量价齐升乏力 / 形态四·量价齐跌衰竭 / 隐性顶背离 / 隐性底背离 / 无形态触发
3. **每行只含信号层信息**：禁止"减仓/加仓/清仓/观望操作"等仓位指令、禁止具体监测价位、禁止 STR/VolSurp/RSI 等指标单列（指标值可作为"信号说明"列的简短证据出现，但不单列）
4. **信号说明 ≤ 25 字**：超出则裁掉冗余表述，只保留 1~2 个最强证据
5. **末尾无扩展章节**：写完最后一只标的 §5.2 摘要后直接结束（或仅含 §6 质量自查输出）

## 5.2 人工阅读输出格式（Markdown 摘要）【扩展节，模板外】

> ⚠️ **适用场景限制**：本节仅适用于**人工 / Coding 工具（如 Cursor）直接调用**本 Skill 的场景。
> Agent 系统流程（`agent.py` 调用 `_call_skill()` → `Signal.from_dict()` 解析）中，**跳过本节，只输出 5 节的标准 JSON**，以避免混合输出导致 JSON 解析失败。
> 若不确定当前是哪种调用场景，**优先只输出 JSON**。

> **人工 / Cursor 调用默认规则**：直接输出汇总速览表 + Markdown 摘要，**不输出 JSON**。
> JSON 仅保留给 Agent 系统调用（`agent.py` / `Signal.from_dict()` 解析），人工文档中省略以保持可读性。

若为人工阅读场景，直接输出以下 Markdown 摘要格式（**省略 §5 的 JSON 块**）：

```markdown

{一句话概括：无形态触发 / 形态名称 + 核心结论}

- 5日累计收益 {±X.X}%，STR ≈ {X.X}（**{平淡/温和/中度/强烈}**，{是否达 ±1.5 中度阈值 / ±2.5 强烈阈值}）

  - **→ 意味着**：{若命中"容量不足/长期不足"，按 §4.4 速查表原文引用；容量充足可省略}
- 价格{XX.XX}元，处于20日区间{位置描述}（高{X}元 / 低{X}元），{新高/新低/无信号}

- 换手率{X.X}%（**{低/中/高/极端}换手分位**，{信号加权 ×1.5 / 未加权 / 剔除}）

- 影线形态：{长上影/长下影/十字星/无显著影线}，上影或下影 ≈ {全日振幅的 X%}（{标准/极端}级反转）
  - **→ 意味着**：{若命中"标准长影线/极端长影线"，按 §4.4 速查表原文引用；非长影线可省略；十字星写"实体过小，多空胶着，等待次日方向选择"}

> **执行约束与监测触发条件**：{A 股制度执行约束（如适用）；或 neutral 信号的量能/价位监测触发条件}
```

**「→ 意味着」子项使用规则**：

| 场景 | 处理 |
|---|---|
| 命中 **强制约束等级**（中度/强烈/爆量/巨量/缩量/地量/低换手/高换手/极端换手/容量不足/长期不足/标准长影线/极端长影线/流动性差/空间确认或未确认） | **必须**写「→ 意味着」子项，引用 §4.4 速查表的「命中逻辑说明」列**原文**；空间位置子项按本节文案直接照写 |
| 命中 **中性等级**（平淡/正常量能/中等换手/容量充足/非长影线） | 「→ 意味着」子项**省略**，避免输出冗余 |
| **多个指标同向共振** | 每个命中指标的「→ 意味着」子项分别写一句，不合并 |
| **指标矛盾**（如 STR 强烈 bearish 但 VolSurp 爆量配合长下影线 bullish） | 各自写「→ 意味着」子项，在「执行约束与监测触发条件」块追加 ⚠️ 标注矛盾来源 |

**各行填写规则**：

| 行 | 来源字段 | 说明 |
|---|---|---|
| 标题行 | `direction` / `confidence` / `meta.risk_level` | direction 用行内代码格式标注；三个信号值用 `/` 分隔 |
| 概括行 | `signals[0]` 或 `reasoning` 首句 | 无形态写"无形态触发，{当前状态}"；有形态写"**形态X（名称）**，{核心数据}" |
| STR 行 | `meta.evidence` STR 值 | **对照 §4.4 指标解读速查表**：平淡 (0~±0.8) / 温和 (±0.8~±1.5) / 中度 (±1.5~±2.5) / 强烈 (>±2.5)；命中温和及以上须附「→ 意味着」子项 |
| VolSurp 行 | `meta.evidence` VolSurp 值 | **对照 §4.4 指标解读速查表**：地量 (<-1.5) / 缩量 (-1~-1.5) / 正常 (-1~+1) / 温和放量 (+1~+2) / 爆量 (+2~+3) / 巨量 (>+3)；命中非"正常量能"须附「→ 意味着」子项 |
| 绝对成交额行 | 行情数据 | **对照 §4.4 过滤器四阈值表**：标注当日成交额、分市场阈值、达标/降权/强制 neutral 状态；命中"容量不足/长期不足"须附「→ 意味着」子项 |
| 价格行 | `meta.evidence` 或行情数据 | 说明价格处于20日高低区间的相对位置，标注是否创新高/新低 |
| 空间位置行 | 行情数据 + 技术位 | **对照 §4.3 支撑位/压力位列表**：标注是否在 MA60 / 周线 MA20 / 前高 / 前低 / 布林上下轨 / 0.618 等关键位 ±3% 内；**必须**附「→ 意味着」子项（确认到位 / 未确认中继风险） |
| 换手率行 | 行情数据 | **对照 §4.4 指标解读速查表**：<1% 低 / 1~5% 中 / >5% 高 / >10% 极端；命中非"中等"须附「→ 意味着」子项 |
| 影线形态行 | 行情数据计算 | **对照 §4.2 长上下影线精确定义**：上影/下影 ≥ 实体 ×2 且 ≥ 振幅 50% 且 振幅 ≥ 昨收 3% 才算长影线；50~70% 标准、>70% 极端；命中长影线或十字星须附「→ 意味着」子项 |
| 执行约束与监测触发条件 | `meta.key_findings` | 引用块格式；含 A 股制度执行约束（如 14:50 前、T+1 缓冲）；若为 `neutral`，需写明转为 bearish/bullish 的量能或价位监测条件 |

**方向标注规范**：

| direction | 标题行显示 | 概括行前缀 |
|---|---|---|
| `bearish` | 🔴 `bearish` | **形态X（形态名称）** 或 **动量衰竭** |
| `neutral` | ⚪ `neutral` | 无形态触发，{状态描述} |
| `bullish` | 🟢 `bullish` | **形态X（形态名称）** 或 **底部反转信号** |

**补充规则**：
- 若 `needs_human_review: true`，在「执行约束与监测触发条件」行后追加：`⚠️ 人工复核：{meta.uncertainties 核心条目}`
- 若触发 A 股涨停板/跌停板特有形态（§4.7），在概括行后追加「**A 股制度约束**：{具体执行时机}」
- `neutral` 的「执行约束与监测触发条件」必须包含「转为 bearish/bullish 的**监测触发条件**（量能条件 + 价位条件）」
- 若**空间位置未确认**（顶背离未撞压力 / 底背离未到支撑），须在概括行后追加「⚠️ 空间位置未确认，警惕{上涨/下跌}中继」，并将该信息同步到 `meta.uncertainties`
- 若**绝对成交额降权或强制 neutral**，须在概括行后追加「⚠️ 绝对成交额不足，VolSurp 信号{降权 / 失效}」
- 若识别为**十字星犹豫**（实体/振幅 < 0.1），direction 强制 neutral，concept 行写"高位/低位十字星，多空胶着，等待次日方向选择"

**输出顺序**：

| 场景 | 输出内容 |
|---|---|
| 单标的，人工调用 | **只输出 Markdown 摘要**（省略 JSON） |
| 多标的，人工调用 | 汇总速览表 → 标的① Markdown 摘要 → 标的② Markdown 摘要 → … |
| Agent 系统调用 | **只输出 JSON**（跳过 5.2 全部内容） |

> ⚠️ **人工调用质量自查（替代 JSON 字段约束）**：输出 Markdown 摘要前，须对照以下两项重点自查——JSON 模式下这两项由字段结构强制填写，Markdown 模式下容易遗漏：
>
> 1. **`uncertainties` 是否写入**：数据缺失（RSI 缺失、历史数据不足）、信号冲突（量价方向与资金流向矛盾）须在「执行约束与监测触发条件」块中以 ⚠️ 标注；
> 2. **`needs_human_review` 是否标注**：凡涉及 RSI 待核实、极端场景边界判断、数据源不完整，须在对应行追加 `⚠️ 人工复核：{具体原因}`。
>
> 完整质量检查见 §6，人工调用时仍须逐项核对 §6 中「仅限人工阅读场景」各条目。

## 6. 质量检查

输出前逐项确认：

- [ ] `direction` 是否明确（bullish / bearish / neutral，无歧义）
- [ ] `confidence` 是否在 0.0~1.0 范围内，是否与信号层数对应
- [ ] `signal_type` 是否为 `technical`
- [ ] `signals` 是否至少有 1 条，包含形态名和关键指标值
- [ ] `meta.evidence` 是否记录了 VolSurp 数值和数据来源
- [ ] `time_horizon` 是否标注为 `short`
- [ ] 强趋势牛市 / 流动性枯竭 / A 股不适用场景是否已处理为 `neutral`
- [ ] **A 股涨停一字板 / 跌停一字板的缩量新高/新低是否已前置排除**（不应判为顶/底背离，须转入 §4.7）
- [ ] **形态三/四的长上下影线是否按精确定义判定**（影线 ≥ 实体 × 2、影线 / 振幅 ≥ 0.5、振幅 / 昨收 ≥ 3% 三条件全成立）；十字星形态是否单独输出 neutral
- [ ] **空间位置确认是否执行**（§4.3 三维度复合）：底背离是否在支撑位附近、顶背离是否在压力位附近；未确认时 confidence 是否显著降权并写入 `meta.uncertainties`
- [ ] **绝对成交额过滤器是否执行**（§4.4 过滤器四）：当日及 20 日均成交额是否对照分市场阈值，降权或强制 neutral 是否落实
- [ ] **confidence 是否对照 §4.4 指标解读速查表逐项累加**，封顶 0.95
- [ ] **每个命中"强制约束等级"的指标是否附带命中逻辑说明**：
  - JSON 模式：`reasoning` 字段是否按「指标名 → 等级 → 命中逻辑」句式拼接 + 每条 `meta.evidence[].note` 是否包含命中等级 + §4.4 速查表的命中逻辑说明原文
  - Markdown 模式：每个命中非中性等级的指标行下方是否有「**→ 意味着**：……」子项，引用 §4.4 速查表的命中逻辑说明原文
- [ ] 中性等级指标（平淡/正常量能/中等换手/容量充足/非长影线）是否**未**写命中逻辑说明，避免冗余
- [ ] 信号冲突是否写入 `meta.uncertainties`
- [ ] 缺失数据是否降低了 `confidence` 并写入 `meta.uncertainties`
- [ ] 是否正确标记 `needs_human_review`
- [ ] 【仅限人工阅读场景】每个标的是否附带了 Markdown 摘要（5.2 节格式）

- [ ] 【仅限人工阅读场景】`neutral` 标的的「执行约束与监测触发条件」是否写明转为 bearish/bullish 的监测触发条件
