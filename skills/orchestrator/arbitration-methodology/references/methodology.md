# 方法论理论依据

> 此文件为 SKILL.md 的理论支撑材料。仲裁引擎和场景设计在需要追溯"为什么权重是这个方向"时按需加载。

## 统一框架：桥水经济机器模型 (Ray Dalio)

市场环境可表达为三个维度的函数：

```
f(经济增长方向, 通胀方向, 风险溢价水平)
```

三种基础场景映射：

| 维度组合 | 场景 | 核心特征 |
|:---|:---|:---|
| 增长↑ + 通胀↓ + 风险溢价压缩 | 牛市（扩张期） | 趋势跟随策略占优 |
| 风险溢价扩张（增长方向可能 ↑ 或 ↓） | 熊市（收缩期） | 风险控制优先于收益获取 |
| 三维度均未形成明确趋势 | 震荡市（均衡期） | 多空力量暂时均衡，等待催化剂 |

## 权重方向推导逻辑

从桥水三维度框架出发，推导每个场景下各分析维度的有效性：

```
桥水框架对该场景的定义
    ↓
该场景下何种分析维度最优效？何种最易失效？
    ↓（参考因子研究、体制分类研究作为验证）
对应专家组的权重方向（提高 / 不变 / 降低）
    ↓
量化数值（待回测校准）
```

权重方向基于理论框架，应保持稳定。
具体数值可在回测中校准，但方向不可因短期实证而翻转。

## 研究方法来源及其角色

以下研究来源并非独立方法论，而是桥水统一框架下的验证与细化材料。

| 研究来源 | 验证内容 | 影响的权重 |
|:---|:---|:---|
| **QuantPedia 因子研究** | 价值/动量因子在牛熊市的收益差异 | 熊市财务面↑ 牛市财务面↓ |
| **LuxAlgo 市场体制** | ADX 趋势/震荡分类，策略匹配验证 | 牛市技术面↑ 震荡市技术面↓ |
| **AQR 动量崩溃** | 熊市反弹中动量策略的最大回撤来源 | 熊市技术面↓↓（仅 RSI<30 恢复） |
| **CEPR 风险定价** | 牛熊市中风险溢价的动态变化 | 牛市风险面↓ 熊市风险面↑↑ |
| **行业轮动 / 林奇 / 护城河** | 行业分析在不同环境下的价值 | 产业分析三种场景均为↑，震荡市最高 |

### 未纳入的研究

牛津新闻情绪、SentimenTrader 情绪追踪、Man Group 多变量模型：这些研究对舆情组和宏观组自身的方法论有参考价值，但对场景选择和权重设计，其结论可合并为"舆情在方向明确时噪声较大（降权），在方向不明时价值有限（降权）"。

## match_signals 规则的理论依据

### 牛市识别（三维度检查）

| 条件 | 对应维度 | 为何以此检查 |
|:---|:---|:---|
| 宏观组 bullish | 经济增长方向 ↑ | 桥水框架中"经济扩张"是牛市的前提——宏观组（专家4组）的专业产出直接回答"经济在扩张吗？" |
| 风控组 bullish | 风险溢价压缩 | 桥水框架中"风险溢价压缩"是牛市的核心特征——风控组（专家7组）的专业产出直接回答"风险在压缩吗？" |
| 整体信号偏多 | 市场共识验证 | 单个 Signal 可能存在噪声，当 7 个专家组形成一致方向时，单一维度判断得到多重交叉验证——这实现了桥水框架中"多个独立来源指向同一方向时可信度最高"的原则 |

### 熊市识别（风险溢价扩张确认）

| 条件 | 对应维度 | 为何以此检查 |
|:---|:---|:---|
| 宏观组 bearish | 经济方向 | 经济增长方向是否支持熊市判断 |
| 风控组 bearish | 风险溢价扩张 | 桥水框架：熊市的本质不是"经济在衰退"，而是"风险溢价在扩张"——这是熊市的核心定义条件 |
| 整体信号偏空 | 市场共识 | 多维度交叉验证，降低单一维度误判风险 |

### 震荡市识别（方向缺失与矛盾检测）

| 条件 | 为何以此检查 |
|:---|:---|
| 宏观组 neutral | 经济方向不明确是震荡市的典型特征——当一个经济体处于"不好不坏"的状态时，多空力量最容易陷入僵持 |
| 多空信号均衡 | 整体信号没有压倒性方向——是桥水框架中"三维度未形成合力"的直接体现 |
| 宏风控信号冲突 | 这是震荡市最强的判别信号——经济看多但风险看空（或反之）意味着市场正经历"好经济但高风险"的矛盾状态，典型震荡市特征 |

## 仓位设计逻辑

仓位配置是独立的风险管理决策：

- **牛市 0.60/0.40**：趋势持续性强，可适度提高仓位以捕捉趋势收益，但始终保留仓位上限（巴菲特"别人贪婪时我恐惧"）
- **熊市 0.25/0.15**：风险溢价扩张阶段，亏损的不可逆性远大于少赚的机会成本——即使7个专家一致看多，熊市中不超过15%
- **震荡市 0.35/0.20**：方向不明时重仓将导致持续小幅损失累积——突破确认后再考虑加仓

## 权重校准方法论

理论锚定 + 实证微调策略：

```
blended = anchor × (1 - α_eff) + empirical × α_eff × scale

α_eff 自适应规则：
  样本 < 10  → α_eff = 0      （完全信任理论）
  样本 < 50  → α_eff = α × 0.5（半信任）
  样本 ≥ 50  → α_eff = α      （正常，默认 α=0.3）
```

约束规则：
- 方向不可逆：理论 ↑ 的不能变为 ↓
- 浮动区间：±30%
- 样本外验证：70% 训练 / 30% 验证
- 稳定性：连续两期波动 < 15%
- 不自动覆盖场景文件
