# 数据管理方案总览（已落地实现）

本文档总结当前仓库 `data_providers/` 下已实现的数据管理方案：统一接口、统一返回格式、Provider 插件化、多市场 symbol 解析、分级缓存、以及财报/公告 PDF 的 raw+normalized/parsed 两套数据集。

## 1. 核心目标

- 让 Agent 只关心“要什么数据、怎么分析”，不关心“数据从哪来、怎么拿、怎么缓存/限频”。
- 对多源数据（量价/财报/公告 PDF/未来宏观与情绪）提供统一入口与统一返回格式。

## 2. 总体架构

- **统一入口（Hub）**：`DataHub` 负责 Provider 路由、限频、缓存、落盘与错误归一化。
  - 位置：`data/hub.py:1`
- **Provider 插件化**：不同数据源实现各自 Provider。
  - 抽象：`data/providers/base.py:1`
- **统一返回格式**：所有取数接口返回 `DataResult(df, meta)`。
  - `df`：`pandas.DataFrame`
  - `meta`：`dataset/source/fetched_at/cached/params`
  - 位置：`data/schemas.py:27`

## 3. 市场与 symbol 规范（已支持 CN/HK/US）

- Agent/调用方可以输入“股票名称/股票代码”。
- `DataHub.resolve_symbol()` 统一归一化为 `Symbol(code,name,market)`，Provider 侧主要用 `code`。
  - 解析器：`data/symbols.py:36`

三市场当前规则：
- **A 股（CN）**：6 位数字（例：`600519`）或名称（例：`贵州茅台`）。
- **港股（HK）**：4/5 位数字，统一补齐为 5 位（例：`0700` → `00700`）。
- **美股（US）**：优先使用东财美股代码（例：`105.MSFT`）；ticker/名称会尝试通过 AkShare 的 spot 列表映射。

## 4. 数据集目录（DatasetType）

数据集统一用 `DatasetType` 标识：`data/schemas.py:11`

### 4.1 量价（当前只做日线）

- `price.ohlcv.daily`：量价日线（OHLCV）
- `price.spot.quote`：实时行情（spot）

Provider：
- `AkShareProvider`（CN/HK/US）：`data/providers/akshare_provider.py:70`

### 4.2 财报（三表）raw + normalized（可选产出）

- `fundamentals.financial_statements.raw`：三表 raw（宽表 + `statement` 标记）
- `fundamentals.financial_statements.normalized`：面向“七步验证链”的常用指标宽表（一行一个报告期）

Provider / Normalizer：
- raw：`EastMoneyProvider`（当前 CN）：`data/providers/eastmoney_provider.py:1`
- normalized：`normalize_financial_statements(raw_df)`：`data/normalizers/financial_statements.py:1`

### 4.3 公告 PDF raw + parsed（可追溯引用）

- `documents.announcements.pdf.raw`：下载 PDF 原文并落盘，返回 `pdf_path/pdf_sha256/...`
- `documents.announcements.pdf.parsed`：逐页解析文本（page 级结果，可直接引用 page）

Provider / Parser：
- raw：`CninfoProvider`（当前 CN）：`data/providers/cninfo_provider.py:1`
- parsed：`parse_pdf_to_pages(pdf_bytes)`：`data/documents/pdf.py:1`

## 5. 统一接口（Agent 怎么用）

说明：所有接口最终返回 `DataResult`，Agent 拿 `res.df` 直接分析。

### 5.1 日线 OHLCV

```python
from data_providers import DataHub, PriceOHLCVRequest

hub = DataHub()
res = hub.get_price_ohlcv_daily(
    PriceOHLCVRequest(symbol="贵州茅台", market="CN", start_date="2024-01-01", end_date="2024-12-31")
)
df = res.df
```

### 5.2 财报 raw + normalized

```python
from data_providers import DataHub, FinancialStatementsRequest

hub = DataHub()
raw = hub.get_financial_statements_raw(FinancialStatementsRequest(symbol="600519", market="CN"))
norm = hub.get_financial_statements_normalized(FinancialStatementsRequest(symbol="600519", market="CN"))
```

### 5.3 公告 PDF raw + parsed（可追溯引用）

```python
from data_providers import DataHub, AnnouncementPDFRequest

hub = DataHub()

    AnnouncementPDFRequest(symbol="600519", market="CN", keyword="年度报告", start_date="2023-01-01", end_date="2025-12-31")

parsed = hub.get_announcement_pdf_parsed(
    AnnouncementPDFRequest(symbol="600519", market="CN", keyword="年度报告", start_date="2023-01-01", end_date="2025-12-31")

# 引用示例：第 3 页文本
```

## 6. 缓存与限频（分级缓存已落地）

- 缓存实现：`DiskCache`（pickle）在 `data/cache.py:1`
- DataHub 默认按 dataset 分级 TTL，并允许 `DataHub(cache_ttls=...)` 覆盖：`data/hub.py:63`

默认 TTL：
- 日线：24h
- 财报 raw/normalized：7d
- 公告 PDF raw/parsed：30d（PDF 同时落盘到 `~/.cache/ai_renaissance/datahub/documents/`）

## 7. 扩展方式（新增数据源/新增数据集）

新增一个数据源：
1. 在 `data/providers/` 新增 Provider 文件并继承 `BaseProvider`
2. 声明 `capabilities`（支持哪些 `DatasetType`）
3. 在 `DataHub` 初始化时注册（或后续建设 ProviderRegistry）

新增一个数据集：
1. 在 `data/schemas.py` 增加 `DatasetType` 枚举项 + Request（如需要）
2. Provider 实现取数 + 最小归一化
3. 如需要统一口径（业务字段宽表），在 `data/normalizers/` 增加 normalized 逻辑
